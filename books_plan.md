# Books 페이지 UI 고도화 및 콘텐츠 변환 지시서 계획

이 계획서는 `books.html` 페이지의 3단 레이아웃(좌측 목차, 중앙 본문, 우측 프로모션)을 고도화하고, 대량의 텍스트 변환 작업을 외부 에이전트(VS Code / 클로드 코드)에 외주(Outsourcing) 주기 위한 전략을 담고 있습니다.

## User Review Required

> [!IMPORTANT]
> **외주 작업(Mission Briefing) 프로세스 승인**
> 중앙 본문의 텍스트 양이 많으므로, 사만다의 쿼타(토큰)를 아끼기 위해 오빠의 제안대로 **"클로드 코드용 초정밀 지시서(Mission Briefing)"**를 만들어 드릴 예정입니다.
> 이 지시서에는 현재 `books.html`의 CSS 클래스(`ch-body`, `callout`, `samantha-say` 등) 규칙이 모두 적혀있게 됩니다.
> 오빠는 그저 로컬에 책 텍스트 파일을 가져다 놓고 이 지시서를 클로드에게 던져주기만 하면 됩니다. 
> 이 방식이 마음에 드시나요?

## Proposed Changes

---

### 1. Left: 목차(TOC) 아코디언 메뉴 구현
- 기존 단순 리스트 형태의 목차를 **접이식(Accordion)** 형태로 변경합니다.
- `<details>` 및 `<summary>` 태그, 또는 커스텀 JS 로직을 사용하여 5개의 챕터를 구성합니다.
- **챕터 1 (무료 공개)**: 클릭 시 하위 목차가 열리며 링크 이동 가능.
- **챕터 2~5 (로그인 전용)**: 기본적으로 자물쇠(🔒) 아이콘 표시. 클릭 시 서브 항목이 보이긴 하지만, 링크를 클릭하면 **회원가입/로그인 유도 모달**이 뜨거나 `join.html`로 리다이렉트되도록 잠금(Locked) 처리.

#### [MODIFY] `books.html` (Left Sidebar)
- `.sidebar-toc` 내부의 아이템들을 계층 구조로 리팩토링.
- 서브 메뉴 트리를 위한 CSS(들여쓰기, 라인 등) 추가.

---

### 2. Center: 변환 작업 지시서(Mission Briefing) 생성
- 방대한 책 텍스트를 HTML 태그로 씌우는 작업을 돕기 위한 가이드 문서를 생성합니다.
- 오빠가 외부 AI(VS Code의 클로드 등)에게 바로 복사+붙여넣기 할 수 있는 마크다운 파일로 제공합니다.

#### [NEW] `mission_briefing_books.md`
- 포함될 내용: 
  - 기본 본문: `<p>`, `<h2>` (`ch-body` 클래스 규칙 적용)
  - 인용구/강조: `<div class="callout">` 
  - 사만다의 말: `<div class="samantha-say">`
  - HTML 적용을 위한 엄격한 규칙 및 예제 템플릿 제공.

---

### 3. Right: 우측 사이드바 위젯 추가
- 빈 공간을 활용하여 추가적인 참여 유도(CTA) 위젯을 배치합니다.

#### [MODIFY] `books.html` (Right Sidebar)
- **방명록 유도 위젯**: "사만다에게 방명록 남기러 가기" 버튼 (아이콘: 📝)
- **다른 책 홍보 위젯**: "Adam의 다른 책 시리즈 미리보기" 위젯 (아이콘: 📚)

## Verification Plan

1. 좌측 목차를 클릭했을 때 서브 항목이 부드럽게 열리고 닫히는지 확인.
2. 잠겨있는 챕터(2~5)를 클릭했을 때 권한 안내가 제대로 동작하는지 확인.
3. 우측 방명록 및 타 도서 홍보 위젯이 글래스모피즘 디자인을 해치지 않고 잘 어울리는지 확인.
4. 오빠가 `mission_briefing_books.md`를 사용해 쉽게 텍스트 변환을 수행할 수 있는지 검증.
