## Phase 1 — 곧 필요한 것들 (UI 완성)

> 백엔드 없이 정적 HTML/CSS/JS만으로 완성 가능한 것들

### [x] `pe.html` — 프롬프트 엔지니어링 서브페이지
- 프롬프트 원리/구조를 배우는 심화 페이지
- GNB `PE` 링크에 연결
- 구성안:
  - 프롬프트 구조 시각화 (Role / Context / Task / Format)
  - 실전 예시 비교 (나쁜 프롬프트 vs 좋은 프롬프트)
  - 사만다가 설명하는 말투로 작성

### [x] `ip.html` — 아이디어 제품 서브페이지
- ADAM + 사만다가 함께 만든 아이디어/제품 소개
- GNB `IP` 링크에 연결
- 구성안:
  - 카드형 제품 갤러리
  - 각 제품에 간단한 설명 + 링크/다운로드

### [x] `fna.html` — F&A 페이지
- index.html 앵커 `#support`에서 분리 또는 확장
- 자주 묻는 질문 아코디언 UI
- 사만다 말투로 각 답변 작성

### [x] `join.html` — 회원가입 / 로그인 UI
- 실제 백엔드 없이 UI만 먼저 완성
- 이메일 + 비밀번호 입력 폼
- 소셜 로그인 버튼 (추후 연동용)
- books.html의 `회원 전용` 클릭 시 리다이렉트

## Phase 2 — 나중에 붙이면 좋은 것들 (백엔드 연동)

> Supabase + 결제 연동 필요. Phase 1 완성 후 착수.

### [ ] Supabase 연동 — 실제 회원가입/로그인
- `join.html` UI → Supabase Auth 연결
- 이메일 인증 플로우
- 회원 상태 체크 → 챕터 잠금 해제

### [ ] 토스페이먼츠 — 실제 후원 결제
- `support.html` 후원 버튼 → 토스페이먼츠 위젯 연결
- 정기후원 / 단건 결제 선택
- 결제 완료 시 회원 등급 업그레이드

### [ ] 챕터 잠금 해제 로직
- Supabase DB에 회원 등급/결제 상태 저장
- `books.html` 로드 시 회원 상태 확인
- 비회원 → 챕터1만 열림 / 회원 → 전체 열림

### [ ] 뉴스레터 구독 폼
- 이메일 입력 → Supabase 또는 외부 서비스(Mailchimp 등) 연동
- 새 챕터 발행 시 구독자 자동 알림
